# 🏛️ Архитектурын шийдвэр — Kodu Sandbox бол ТУСДАА үйлчилгээ

> Шийдсэн: 2026-07-17, Zulzaga. Статус: **баталсан** ✅

## Шийдвэр

**Kodu Sandbox-ыг KoDu-гийн кодтой шууд холихгүй.**
Оронд нь **тусдаа сайт/үйлчилгээ** болгож, **API** гаргаж, KoDu түүн рүү
зөвхөн API-аар ханддаг байна.

```
Одоо:    KoDu → (API) → E2B
Ирээдүй: KoDu → (API) → Kodu Sandbox  ← тусдаа сервер, тусдаа repo, тусдаа сайт
```

## Яагаад

1. **E2B-ийн хэв маягийг давтана.** E2B өөрөө тусдаа үйлчилгээ + API.
   KoDu аль хэдийн "гадаад sandbox API дуудах" бүтэцтэй тул шилжихдээ
   **зөвхөн хаяг/клиентээ** солино — дотоод кодоо их ухахгүй.
2. **Аюулгүй байдлын дүрэм №1** (master-plan §3): sandbox сервер дээр QPay
   түлхүүр, DB нууц үг, хэрэглэгчийн өгөгдөл **байж болохгүй.** Тусдаа
   үйлчилгээ болгосноор энэ дүрэм өөрөө биелнэ.
3. **Ирээдүйд бие даасан бүтээгдэхүүн** болох замтай (roadmap-ийн
   "Kodu Cloud"). Тусдаа API-тай үйлчилгээг хожим бусдад ч нээж болно.
4. **Туршилт, эвдрэл тусгаарлагдана.** Sandbox унасан ч KoDu-гийн үндсэн
   сайт, төлбөр, хэрэглэгчийн өгөгдөл огт өртөхгүй.

## Юу гэсэн үг вэ (практикт)

| Тал | Юу байх вэ |
|-----|-----------|
| **Энэ repo (SAND)** | Бие даасан үйлчилгээ: controller API + template images + (хожим) өөрийн жижиг сайт/самбар |
| **API** | Одоо байгаа: `POST/GET/DELETE /api/previews`. Дараа: **API түлхүүр** (auth), rate limit, файлын хэмжээний хязгаар |
| **Байршил** | Хожим тусдаа VPS дээр (KoDu-гийн серверээс өөр) |
| **KoDu-гийн тал** | Зөвхөн жижиг client код: одоогийн E2B client-тэй ижил interface-тэй "Kodu Sandbox client". KoDu-гийн repo-д sandbox-ийн логик **орохгүй** |

## Дараагийн алхмууд (энэ шийдвэрээс урган гарах)
