# 🐳 Kodu Sandbox — Master Plan

> Найз (KoDu) + Zulzaga-гийн хамтын урт хугацааны төлөвлөгөө.
> Зарчим: **өдөр бүр 1%. Нэг асуудал → нэг шийдэл → дараагийнх. Бид яарахгүй.**

---

## 0. 🎯 Эцсийн зорилго (North Star)

KoDu одоо хэрэглэгчийн сайтыг **E2B** дээр preview хийдэг (гуравдагч этгээдийн cloud).
Урт хугацаанд бид **өөрийн sandbox** — **Kodu Sandbox** — эзэмшинэ.

```
Одоо:    Kodu → E2B → Preview
Ирээдүй: Kodu → Kodu Sandbox → Preview
```

**Гэхдээ хамгийн чухал стратеги — ЗЭРЭГЦЭЭ 2 ЗАМ:**

1. 🟢 **E2B бодит хэрэглэгчдэд үйлчилсээр байна** (амьд бүтээгдэхүүнээ эрсдэлд оруулахгүй)
2. 🔨 Түүний **зэрэгцээ** бид Docker сурч, Kodu Sandbox **v1** барина
3. ✅ v1 нь **баталгаажиж, аюулгүй болсон** үед л E2B-г солино

> Амьд бүтээгдэхүүнээ бариу гэхэд, доор нь ирээдүйгээ угсарна. Хуучин хоолоо иднэ, шинэ гал тогоо барина.

## 1. 🧠 Sandbox гэж юу вэ (энгийнээр)

**Docker** = таны компьютер дотор ажилладаг **жижиг компьютерууд**.

```
Таны сервер
│
├── Container #1  → Ubuntu + Node + Next.js   (Хэрэглэгч А-гийн сайт)
├── Container #2  → Ubuntu + Node + Next.js   (Хэрэглэгч Б-гийн сайт)
└── Container #3  → Ubuntu + Node + Next.js   (Хэрэглэгч В-гийн сайт)
```

- Container бүр **өөрийн жижиг ертөнцтэй** — нэгнийх нь алдаа нөгөөд нөлөөлөхгүй.
- Хэрэглэгч зөвхөн **Preview** харна. Доор нь Docker байгааг мэдэхгүй.
- Ажил дуусмагц container-ыг **устгана** → RAM, CPU чөлөөлөгдөнө.

**Энэ чинь яг E2B хийдэг ажил.** Бид үүнийг өөрийн сервер дээр хийнэ.

## 2. 🏗️ Архитектур (одоо → ирээдүй)

### v1 (энэ жилийн зорилго) — зөвхөн Docker
```
Хэрэглэгч → Kodu App → Kodu Sandbox Controller (Node + dockerode)
                              │
                              ▼
                      docker run (Ubuntu + Node)
                              │
                       npm run dev → :3000
                              │
                       Preview URL ← хэрэглэгч рүү
                              │
                       TTL дуусмагц → docker rm
```

### v2+ (дараа жил) — жинхэнэ аюулгүй тусгаарлалт
- Жирийн Docker → **gVisor (runsc)** эсвэл **Firecracker microVM** руу шатлана
- (Учир нь жирийн Docker нь итгэлгүй кодын эсрэг хангалттай хана биш — доор "Аюулгүй байдал" хэсгийг үз)

### Chrome-ий roadmap
```
2026:  Kodu Builder ✅ → Kodu Preview ✅ → Kodu Sandbox v1
2027:  Kodu Sandbox v2 → Kodu Auth ✅ → Kodu Deploy
2028:  Kodu Cloud → "Deploy to Kodu"
2029:  Kodu Infrastructure
```

## 3. 🔒 Аюулгүй байдлын дүрэм (ХАМГИЙН ЧУХАЛ ХЭСЭГ)

> Docker сурах нь амархан. Хэцүү нь: **олон хүний итгэлгүй кодыг нэг сервер дээр аюулгүй тусгаарлах.**

**Алтан дүрэм:**
1. 🚫 Sandbox сервер дээр **QPay түлхүүр / DB нууц үг / хэрэглэгчийн өгөгдөл БАЙЖ БОЛОХГҮЙ.** Sandbox бол Kodu App серверээс **тусдаа** сервер байх ёстой.
2. 🧊 Container бүр **түр зуурын (ephemeral)** — дуусмагц устана.
3. 👤 Container доторх код **root биш** хэрэглэгчээр ажиллана (`--user`).
4. 📉 Нөөцийн хязгаар заавал: `--memory=512m --cpus=1 --pids-limit=256`.
5. 🌐 Сүлжээний гарцыг хязгаарла (хортой код интернэтээр рүү дайрахаас сэргийлэх).
6. 📁 Host-ийн эмзэг фолдерыг container руу mount **хийхгүй**.
7. 🔐 v2 дээр **gVisor / microVM** — жинхэнэ escape-ээс хамгаалах давхарга.

**v1 сурах үеийн алтан дүрэм:** сурч байх хугацаандаа **бодит хэрэглэгчийн кодыг нууц түлхүүртэй серверт битгий ойртуул.** Тусдаа туршилтын сервер дээр сур.

## 4. 📚 Суралцах зам (өдөрт 2 цаг) — ~30–45 хоног

> Зорилго: чи **инженер болох биш.** Зорилго: Docker-ыг **ойлгож, AI-г (Claude Code / Codex) чиглүүлж, ажиллаж байгааг батлах** хэмжээнд сурах. Яг л KoDu-г бариж байгаа шигээ.

### 1-р долоо хоног — Docker үндэс
- Docker гэж юу вэ? Image vs Container vs Volume vs Port гэдгийг **ойлгох**
- Дадлага: `docker run hello-world`, `docker run -it ubuntu bash`, `docker ps`, `docker images`, `docker rm`
- ✅ Мэдсэн шинж: ubuntu container асааж, дотор нь орж, устгаж чадвал болоо

### 2-р долоо хоног — Linux / Ubuntu команд
- `cd  ls  mkdir  rm  cp  mv  cat  nano` — фолдер, файлаар зөрөх
- ✅ Мэдсэн шинж: container дотор фолдер үүсгэж, файл бичиж, харж чадвал болоо

### 3-р долоо хоног — Node Docker дотор
- `node`, `npm install`, `npm run dev`, (`pnpm`, `bun`)
- Дадлага: ubuntu container дотор Node суулгаад жижиг script ажиллуул
- ✅ Мэдсэн шинж: container дотор `node -v` ажиллавал болоо

### 4-р долоо хоног — Next.js-ийг Docker дотор ажиллуулах ⭐
- `Dockerfile` бичих → `docker build -t myapp .` → `docker run -p 3000:3000 myapp`
- Браузер дээр `localhost:3000` нээгдвэл...
- 🎉 **Чи Sandbox-ийн 30%-ийг хийчихлээ!**

### 5–8-р долоо хоног — Кодоор Docker удирдах (Preview Engine-ий цөм)
- **`dockerode`** (Node-ийн Docker SDK) — код бичээд container **асаах/унтраах**
- Нэг хүсэлт ирэхэд → container үүсгэ → файл бич → `npm run dev` → port нээ → Preview URL буцаа → TTL дараа устга
- ✅ Мэдсэн шинж: товч дархад код нэг container асааж, preview буцааж, дараа нь устгаж чадвал → **Kodu Sandbox v1 болсон!**

## 5. 🚀 Build Roadmap — Kodu Sandbox v1 (6–12 сар, бага багаар)

| Үе шат | Юу хийх | Хугацаа |
|--------|---------|---------|
| **M0** | Docker сурах (4-р хэсэг) | 1-р сар |
| **M1** | Кодоор **нэг** container асааж preview харуулах | 2-р сар |
| **M2** | **Хэрэглэгч бүрт** container + lifecycle + автомат устгал | 3-р сар |
| **M3** | Preview URL routing + дулаан pool (хурд) | 4-р сар |
| **M4** | **Аюулгүй байдал** чангатгах (нөөц, сүлжээ, non-root) | 5-р сар |
| **M5** | E2B-ийн **зэрэгцээ** shadow-тест (жинхэнэ солихгүйгээр) | 6-р сар |
| **M6** | Эрсдэл багатай preview-г аажим **шилжүүлэх**, E2B fallback | 7–9-р сар |
| **v2** | gVisor / microVM, Kodu Cloud, Kodu Deploy | 2027 |

## 6. ⚡ Preview Engine (яаж ажиллах вэ)

```
1. Хэрэглэгч сайт хийнэ
2. Controller → чөлөөт container авна (эсвэл шинээр асаана)
3. Кодын файлуудыг container руу бичнэ
4. npm run dev ажиллуулна
5. Container-ий портыг гадагш нээж Preview URL үүсгэнэ
6. Хэрэглэгч рүү URL буцаана
7. TTL (жишээ 15 мин) дуусмагц container-ыг устгана
```

**Хурдны нууц:** урьдчилан асаасан **дулаан container-ий pool** барь → хэрэглэгч хүлээхгүй.
