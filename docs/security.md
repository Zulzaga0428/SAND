# 🔒 Kodu Sandbox — Аюулгүй байдлын загвар

> Sandbox-д хэрэглэгчийн **итгэлгүй код** ажиллана. Тиймээс "олон давхар хана"
> зарчмаар хамгаална. KoDu-тай холбохоос ӨМНӨ энэ бүхэн бат байх ёстой.

## Давхаргууд (defense in depth)

| # | Хамгаалалт | Хэрхэн | Юунаас хамгаална |
|---|-----------|--------|------------------|
| 1 | **Root биш** | `User: node` (uid 1000) | Систем эзэмших |
| 2 | **Capability бүгд хасах** | `CapDrop: ["ALL"]` | raw socket, mount, module ачаалах г.м |
| 3 | **no-new-privileges** | `SecurityOpt` | setuid-ээр эрх нэмэгдүүлэх |
| 4 | **Нөөцийн хязгаар** | Memory, NanoCpus, PidsLimit, swap=0 | RAM/CPU дуусгах, fork bomb |
| 5 | **Ephemeral** | `AutoRemove` + 15 мин TTL | Байнгын халдлага, үлдэгдэл |
| 6 | **Контейнер хооронд тусгаарлалт** | `enable_icc=false` | Нэг preview нөгөөд халдах |
| 7 | **Egress firewall** | iptables DOCKER-USER (NEW drop) | Интернэт рүү дайрах, метадата хулгайлах, host руу орох |
| 8 | **API түлхүүр + rate limit** | Bearer auth, IP-д 30/мин | Зөвшөөрөлгүй хандалт, DoS |
| 9 | **Оролтын хязгаар** | ≤50 файл, ≤512KB/файл, ≤2MB нийт | Диск/санах ой дүүргэх |

## Мастер төлөвлөгөөний дүрэмтэй харьцуулбал

master-plan §3-ын 7 алтан дүрэм:
- ✅ №1 Sandbox дээр нууц түлхүүр байхгүй (тусдаа сервер, зөвхөн sandbox)
- ✅ №2 Ephemeral
- ✅ №3 root биш
- ✅ №4 нөөцийн хязгаар
- ✅ №5 сүлжээний гарц хязгаарлагдсан (egress firewall)
- ✅ №6 host фолдер mount хийхгүй
- ⬜ №7 gVisor/microVM — **v2** (жинхэнэ kernel escape-ээс хамгаалах давхарга)

## Хэрхэн суулгаж, батлах вэ (VPS дээр)

```bash
# 1. Egress firewall суулгах (нэг удаа)
sudo bash /opt/kodu-sandbox/deploy/harden.sh

# 2. Халдлагын туршилт — бүгд PASS байх ёстой
sudo bash /opt/kodu-sandbox/deploy/redteam-test.sh
```

`redteam-test.sh` нь "хакер" болж дараахыг оролдоно (бүгд ХААГДСАН = PASS):
1. Интернэт рүү гарах
2. Cloud метадата хулгайлах (169.254.169.254)
3. Capability үлдсэн эсэх
4. no-new-privileges идэвхгүй эсэх
5. Контейнер хооронд ping

## Одоогийн хязгаарлалт (мэдэж байх ёстой)

- **Egress бүрэн хаалттай:** одоогоор preview доторх сайт гадны API дуудаж
  чадахгүй. Хэрэгтэй бол хожим allowlist (зөвшөөрсөн домэйн) нэмж болно.
- **Жирийн Docker (v1):** kernel-ийн гүнзгий цоорхойгоос 100% хамгаалахгүй.
  Production-д олон хэрэглэгчтэй болоход **gVisor эсвэл microVM (v2)** руу шатлана.
  Одоохондоо: итгэлгүй кодыг **зөвхөн энэ тусдаа, нууцгүй серверт** ажиллуулна.
